# Лабораторная работа №9: Модульное тестирование (Unit Testing)

## Новые технологии и концепции

### 1. Модульное тестирование (Unit Testing)
- Процесс проверки отдельных модулей исходного кода программы (обычно методов или классов) на корректность работы.
- **xUnit**: Популярный фреймворк для модульного тестирования в .NET.
- **AAA (Arrange, Act, Assert)**: Паттерн написания тестов:
  - **Arrange**: Подготовка данных и объектов.
  - **Act**: Выполнение тестируемого действия.
  - **Assert**: Проверка результата.

### 2. Атрибуты xUnit
- `[Fact]`: Тест, который всегда верен (инвариант). Не принимает параметров.
- `[Theory]`: Тест, который верен для определенного набора данных.
- `[InlineData]`: Передает набор данных непосредственно в атрибуте.
- `[ClassData]`, `[MemberData]`: Позволяют использовать классы или методы как источники тестовых данных.

### 3. Имитация зависимостей (Mocking)
- **NSubstitute**: Библиотека для создания имитаций (mock-объектов) интерфейсов и классов. Позволяет подменять реальные зависимости (например, базу данных или внешний сервис) на контролируемые заглушки.
- Позволяет задавать поведение методов (`Returns`) и проверять их вызовы (`Received`).

### 4. Тестирование с базой данных
- **SQLite In-Memory**: Использование легковесной базы данных SQLite в оперативной памяти для тестов, требующих контекста EF Core. Это быстрее и надежнее, чем использование реальной БД, и точнее, чем `InMemory` провайдер EF Core.
