# Лабораторная работа №4: Работа с REST API

## Новые технологии и концепции

### 1. REST API (Representational State Transfer)
- Архитектурный стиль взаимодействия компонентов распределенного приложения в сети.
- Использует HTTP методы (GET, POST, PUT, DELETE) для выполнения операций над ресурсами.
- **ASP.NET Core Web API**: Фреймворк для создания RESTful сервисов.

### 2. Entity Framework Core (EF Core) и PostgreSQL
- **EF Core**: Объектно-реляционный маппер (ORM), позволяющий работать с базой данных через объекты C#.
- **PostgreSQL**: Мощная объектно-реляционная система управления базами данных с открытым исходным кодом.
- **Миграции**: Механизм EF Core для синхронизации схемы базы данных с моделями данных в коде (`dotnet ef migrations add`, `dotnet ef database update`).

### 3. Minimal API
- Новый подход в ASP.NET Core для создания HTTP API с минимальным количеством кода и настроек.
- Позволяет определять маршруты и обработчики прямо в `Program.cs` (или методах расширения) без создания классов контроллеров.
- Использует методы `MapGet`, `MapPost`, `MapGroup` и т.д.

### 4. Паттерн CQRS и MediatR
- **CQRS (Command Query Responsibility Segregation)**: Паттерн, разделяющий операции чтения (Query) и операции изменения данных (Command).
- **MediatR**: Библиотека, реализующая паттерн "Посредник" (Mediator). Позволяет уменьшить связность кода, отправляя запросы (Request) и команды обработчикам (Handlers) через посредника, вместо прямого вызова.

### 5. Swagger / OpenAPI
- Инструмент для описания, документирования и тестирования RESTful веб-сервисов.
- Генерирует интерактивную документацию, позволяющую отправлять запросы к API прямо из браузера.
