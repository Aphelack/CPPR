# Лабораторная работа №3: Работа с данными

## Новые технологии и концепции

### 1. Разделение на слои (Domain Layer)
- Выделение сущностей и моделей в отдельную библиотеку классов (`.Domain`). Это позволяет использовать одни и те же классы в разных частях приложения (например, в API и в UI) и избегать циклических зависимостей.

### 2. Паттерн "Сервис" (Service Pattern)
- Вынесение бизнес-логики и логики доступа к данным из контроллеров в отдельные классы-сервисы.
- Контроллеры становятся "тонкими", отвечая только за прием запроса и выбор представления.

### 3. Внедрение зависимостей (Dependency Injection - DI)
- Механизм, позволяющий объектам получать свои зависимости извне, а не создавать их самостоятельно.
- В ASP.NET Core встроен DI-контейнер. Сервисы регистрируются в `Program.cs` (или через методы расширения) с определенным временем жизни:
    - **Transient**: Создается каждый раз при запросе.
    - **Scoped**: Создается один раз на HTTP-запрос.
    - **Singleton**: Создается один раз на все время работы приложения.

### 4. Асинхронное программирование
- Использование ключевых слов `async` и `await`, а также типа `Task<T>`.
- Позволяет не блокировать поток выполнения во время длительных операций (например, запрос к базе данных или чтение файла), что повышает производительность веб-сервера.

### 5. Модели-обертки (ResponseData, ListModel)
- **ResponseData<T>**: Универсальный класс для стандартизации ответов сервисов. Содержит данные, флаг успеха и сообщение об ошибке.
- **ListModel<T>**: Класс для передачи списков данных с информацией о пагинации (текущая страница, всего страниц).

### 6. Пагинация и фильтрация
- **Skip/Take**: Методы LINQ для пропуска определенного количества элементов и выборки следующей порции (для пагинации).
- **Where**: Метод LINQ для фильтрации данных по условию.

### 7. Конфигурация (appsettings.json)
- Хранение настроек приложения (например, количество элементов на странице) в JSON-файле.
- Доступ к настройкам через интерфейс `IConfiguration`.
