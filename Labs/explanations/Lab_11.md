# Лабораторная работа №11: Введение в Blazor

## Новые технологии и концепции

### 1. Blazor
- **Blazor**: Фреймворк от Microsoft для создания интерактивных веб-интерфейсов с использованием C# вместо JavaScript.
- **Razor Components**: Основные строительные блоки Blazor. Это файлы с расширением `.razor`, содержащие HTML-разметку и код C#.
- **Blazor Server (Interactive Server)**: Модель хостинга, при которой приложение выполняется на сервере. Обновления UI отправляются клиенту через постоянное соединение **SignalR** (обычно WebSockets). Браузер выступает как "тонкий клиент", просто отображая изменения DOM.

### 2. Компоненты и маршрутизация
- **App.razor**: Корневой компонент приложения.
- **Routes.razor**: Компонент, отвечающий за маршрутизацию (выбор отображаемого компонента в зависимости от URL).
- **NavLink**: Компонент для создания навигационных ссылок, который автоматически добавляет CSS-класс `active`, если ссылка соответствует текущему URL.
- **@Body**: Директива в макете (Layout), указывающая место, куда будет подставлено содержимое конкретной страницы.

### 3. Формы и валидация
- **EditForm**: Встроенный компонент Blazor для работы с формами. Поддерживает валидацию модели.
- **InputNumber**, **InputText**: Компоненты ввода, привязываемые к свойствам модели.
- **DataAnnotationsValidator**: Компонент, включающий валидацию на основе атрибутов данных (Data Annotations), таких как `[Required]`, `[Range]`.
- **ValidationSummary**: Выводит список всех ошибок валидации.

### 4. Жизненный цикл и параметры
- **[Parameter]**: Атрибут, позволяющий передавать данные в компонент извне (от родителя или из URL).
- **OnInitialized / OnInitializedAsync**: Метод жизненного цикла, вызываемый при инициализации компонента.
- **OnParametersSet / OnParametersSetAsync**: Вызывается при изменении параметров компонента.
