# Лабораторная работа №1: Знакомство с приложением ASP.NET Core

## Новые технологии и концепции

### 1. Структура проекта ASP.NET Core MVC
- **Controllers**: Классы, обрабатывающие входящие HTTP-запросы. Они содержат методы действий (Actions), которые возвращают результат (например, представление).
- **Views**: Файлы с расширением `.cshtml`, содержащие HTML-разметку и код C# (Razor). Они отвечают за отображение данных пользователю.
- **wwwroot**: Корневая папка для статических файлов (CSS, JavaScript, изображения). Файлы в этой папке доступны напрямую через HTTP.

### 2. Razor Views
- **Razor**: Синтаксис шаблонизации, позволяющий внедрять код C# в HTML. Код начинается с символа `@`.
- **_Layout.cshtml**: Файл макета, определяющий общую структуру страниц (шапку, подвал, подключение стилей и скриптов). Позволяет избежать дублирования кода.
- **_ViewStart.cshtml**: Файл, который выполняется перед запуском любого представления. Обычно используется для установки свойства `Layout` по умолчанию.
- **_ViewImports.cshtml**: Файл для подключения пространств имен и Tag Helpers, которые будут доступны во всех представлениях.
- **@RenderBody()**: Метод, вызываемый в макете, который указывает место, куда будет вставлено содержимое конкретного представления.
- **Sections**: Механизм (`@RenderSection`, `@section`), позволяющий представлениям вставлять контент в определенные места макета (например, скрипты в конец `<body>`).

### 3. Bootstrap
- CSS-фреймворк для создания адаптивных и стильных веб-интерфейсов.
- Использует сеточную систему (Grid System) для расположения элементов.
- Предоставляет готовые компоненты (Navbar, Dropdown, Card и т.д.).

### 4. HTTP Методы
- **GET**: Используется для запроса данных. Параметры передаются в строке запроса (URL).
- **POST**: Используется для отправки данных на сервер (например, формы). Данные передаются в теле запроса.
