# Ответы на вопросы для самопроверки (Лабораторная работа №11)

### 1. Что такое компоненты Blazor?
Это самодостаточные модули пользовательского интерфейса (файлы `.razor`), которые могут содержать разметку (HTML), логику (C#) и стили. Они могут быть вложенными, повторно используемыми и могут обрабатывать события.

### 2. Как передаются данные между клиентом и сервером, если установлен атрибут `@rendermode InteractiveServer`?
Данные передаются через постоянное соединение **SignalR** (обычно по протоколу WebSockets) в бинарном формате. Сервер отправляет "разностные" обновления DOM (diffs), а клиент отправляет события UI (клики, ввод текста).

### 3. Как передать данные компоненту?
Через свойства компонента, помеченные атрибутом `[Parameter]`.
```html
<MyComponent Title="Hello" Count="10" />
```

### 4. Как установить привязку данных в компоненте Blazor?
Используя директиву `@bind` или `@bind-Value`.
```html
<input @bind="currentValue" />
```

### 5. Когда возникает событие `OnParameterSetAsync`?
Оно вызывается после инициализации компонента (`OnInitialized`) и каждый раз, когда родительский компонент передает новые значения параметров.

### 6. Как обработать событие «click» кнопки компонента Blazor?
Используя атрибут `@onclick`.
```html
<button @onclick="IncrementCount">Click me</button>
```

### 7. Когда и для чего используется атрибут `@attribute [StreamRendering(true)]`?
Используется при SSR (Server-Side Rendering) для потоковой передачи HTML. Это позволяет серверу отправить начальную разметку (например, "Загрузка...") клиенту сразу, не дожидаясь завершения длительных асинхронных операций, а затем "дослать" обновленный контент в тот же HTTP-ответ.

### 8. Когда выполняется привязка данных поля `<input>` к свойству в коде компонента Blazor?
По умолчанию — при потере фокуса (событие `onchange`). Можно изменить это поведение на "при вводе" (событие `oninput`), используя `@bind:event="oninput"`.

### 9. Для чего используется компонент `Router`?
Он анализирует текущий URL браузера и определяет, какой компонент (страницу) нужно отобразить. Если маршрут найден, он рендерит компонент `Found`, иначе — `NotFound`.

### 10. Как в компоненте Blazor получить значение сегмента маршрута к странице?
Определить свойство с атрибутом `[Parameter]` и тем же именем, что и параметр в маршруте (`@page "/product/{id}"`).
```csharp
[Parameter] public string Id { get; set; }
```

### 11. Для чего используется класс `NavigationManager`?
Для программного управления навигацией (перенаправление пользователя) и работы с URI (получение текущего адреса, базового адреса).
```csharp
Navigation.NavigateTo("/home");
```

### 12. На странице (родительский компонент) размещен компонент (дочерний компонент). Как передать данные от дочернего компонента к родительскому?
Через `EventCallback`. Дочерний компонент объявляет параметр типа `EventCallback<T>`, а родительский подписывается на него.
```csharp
// Child
[Parameter] public EventCallback<int> OnCountChanged { get; set; }
// Parent
<Child OnCountChanged="HandleChange" />
```
