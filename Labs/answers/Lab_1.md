# Ответы на вопросы для самопроверки (Лабораторная работа №1)

1. **В какой папке проекта хранятся файлы представлений контроллера?**
   В папке `Views`, внутри подпапки с именем контроллера (например, `Views/Home` для `HomeController`).

2. **Где в проекте должен находиться статический контент?**
   В папке `wwwroot`.

3. **Где в проекте создается конвейер обработки запросов?**
   В файле `Program.cs`.

4. **Где в проекте размещаются файлы классов контроллеров?**
   В папке `Controllers`.

5. **Для чего используется файл `_ViewStart.cshtml`?**
   Для выполнения кода перед рендерингом каждого представления. Обычно используется для установки свойства `Layout` по умолчанию для всех представлений.

6. **Для чего используется файл `_ViewImports.cshtml`?**
   Для подключения пространств имен (`using`) и Tag Helpers (`@addTagHelper`), которые должны быть доступны во всех файлах представлений, чтобы не прописывать их в каждом файле отдельно.

7. **Для чего используется файл макета (Layout)?**
   Для определения общей структуры веб-страниц (header, footer, меню, подключение CSS/JS), чтобы избежать дублирования кода на каждой странице.

8. **Как в Layout указать место, куда будет помещена разметка представления?**
   С помощью вызова метода `@RenderBody()`.

9. **Что такое «Section» на макете?**
   Это именованная область в макете, куда конкретное представление может вставить свой контент (например, специфичные для страницы скрипты или стили). Определяется с помощью `@RenderSection("Name", required: boolean)`.

10. **Как использовать секцию в представлении?**
    С помощью директивы `@section Name { ... }`.

11. **Как указать, что представление использует Layout?**
    Установить свойство `Layout` в блоке кода представления: `@{ Layout = "_LayoutName"; }`. Либо это делается автоматически через `_ViewStart.cshtml`.

12. **Как указать, что представление не использует Layout?**
    Установить `Layout = null;` в блоке кода представления.

13. **Что означает утверждение: «HTTP не поддерживает сохранение состояний»?**
    Это означает, что сервер не запоминает информацию о предыдущих запросах от того же клиента. Каждый запрос обрабатывается независимо, без контекста предыдущих взаимодействий (stateless protocol).

14. **Как в form указать адрес отправки данных?**
    С помощью атрибута `action` (например, `<form action="/Home/Create">`) или Tag Helper `asp-action` / `asp-controller`.

15. **В каком виде данные передаются на сервер?**
    В виде пар "ключ-значение". При GET-запросе — в строке запроса (Query String), при POST-запросе — в теле запроса (Request Body).

16. **Разница между POST и GET?**
    - **GET**: Параметры в URL, данные видны пользователю, ограниченный объем данных, кэшируется браузером, идемпотентный (не меняет состояние сервера).
    - **POST**: Параметры в теле запроса, данные скрыты из адресной строки, нет жесткого ограничения по объему, не кэшируется, используется для изменения данных на сервере.
