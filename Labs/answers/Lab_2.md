# Ответы на вопросы для самопроверки (Лабораторная работа №2)

1. **Для чего используется tag-helper `asp-action`?**
   Для генерации URL-адреса, указывающего на определенный метод действия (Action) контроллера. Используется в тегах `<a>` (атрибут `href`) и `<form>` (атрибут `action`).

2. **Что такое частичное представление?**
   Это файл представления Razor (обычно с расширением `.cshtml`), который содержит фрагмент HTML-разметки и может быть встроен в другие представления. Используется для повторного использования кода разметки.

3. **Как разместить частичное представление в разметке представления?**
   - С помощью Tag Helper: `<partial name="_PartialName" />`
   - С помощью HTML Helper: `@await Html.PartialAsync("_PartialName")`

4. **Чем компонент представления отличается от частичного представления?**
   Частичное представление содержит только разметку и получает данные от родительского представления. Компонент представления имеет собственный класс с логикой (C#), который может самостоятельно получать и обрабатывать данные перед рендерингом.

5. **Как разместить компонент представления в разметке?**
   - С помощью метода: `@await Component.InvokeAsync("ComponentName", parameters)`
   - С помощью Tag Helper (в новых версиях): `<vc:component-name param="value" />`

6. **Где находится файл представления компонента?**
   Обычно по пути: `Views/Shared/Components/{ComponentName}/Default.cshtml` или `Views/{ControllerName}/Components/{ComponentName}/Default.cshtml`.

7. **Чем отличаются ViewData и ViewBag?**
   `ViewData` — это словарь (`Dictionary<string, object>`), требующий приведения типов при извлечении данных. `ViewBag` — это динамическая обертка (`dynamic`) над `ViewData`, позволяющая обращаться к данным через свойства без явного приведения типов.

8. **Что такое модель представления?**
   Это объект (обычно C# класс), который контроллер передает в представление для отображения данных. Представление может быть строго типизировано этой моделью.

9. **Как указать в представлении класс модели?**
   С помощью директивы `@model Namespace.ClassName` в начале файла представления.

10. **Как получить доступ к членам модели представления?**
    С помощью свойства `@Model` (например, `@Model.Name`, `@Model.Id`).
