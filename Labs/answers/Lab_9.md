# Ответы на вопросы для самопроверки (Лабораторная работа №9)

### 1. Для чего используют модульные тесты?
Для проверки корректности работы отдельных модулей (методов, классов) изолированно от остальной системы. Это помогает находить ошибки на ранних стадиях, упрощает рефакторинг и служит документацией к коду.

### 2. Что означает шаблон ААА в модульном тестировании?
- **Arrange (Подготовка)**: Настройка тестового окружения, создание объектов, моков и входных данных.
- **Act (Действие)**: Вызов тестируемого метода.
- **Assert (Проверка)**: Проверка того, что результат действия соответствует ожиданиям.

### 3. Для чего используется библиотека NSubstitute?
Для создания имитаций (mock-объектов) зависимостей. Это позволяет тестировать класс в изоляции, подменяя сложные или внешние зависимости (например, доступ к БД или API) на простые объекты с заданным поведением.

### 4. Как в XUnit проверить, что значение типа int, возвращенное тестируемым методом, верно?
Используя `Assert.Equal(expected, actual)`.
```csharp
Assert.Equal(5, result);
```

### 5. Как протестировать значение, передаваемое во ViewData?
Нужно привести результат действия к типу `ViewResult` и проверить его свойство `ViewData`.
```csharp
var viewResult = Assert.IsType<ViewResult>(result);
Assert.Equal("ExpectedValue", viewResult.ViewData["Key"]);
```

### 6. Как в XUnit проверить, что метод генерирует исключение?
Используя `Assert.Throws<ExceptionType>(action)`.
```csharp
Assert.Throws<ArgumentNullException>(() => service.Method(null));
```

### 7. Как в NSubstitute создать имитацию метода класса?
Нужно создать мок интерфейса или виртуального метода класса и настроить его поведение с помощью метода расширения `Returns`.
```csharp
var mockService = Substitute.For<IService>();
mockService.GetData().Returns("MockData");
```
