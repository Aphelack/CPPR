# Ответы на вопросы для самопроверки (Лабораторная работа №12)

### 1. В какой вид компилируется клиентский код приложения Blazor WebAssembly?
Код C# компилируется в стандартные DLL-сборки .NET. В браузере эти DLL загружаются и выполняются средой выполнения .NET (dotnet.wasm), которая сама скомпилирована в WebAssembly.

### 2. Чем отличается одностраничное приложение (SPA) от обычного веб-приложения?
В SPA (Single Page Application) страница загружается один раз. Дальнейшее взаимодействие с пользователем происходит через динамическое обновление содержимого страницы с помощью JavaScript (или WebAssembly) и AJAX-запросов к API, без полной перезагрузки страницы. Обычное приложение перезагружает страницу при каждом переходе.

### 3. Что такое CascadingParameter и как его использовать?
Это параметр, который передается от родительского компонента всем его потомкам (прямым и непрямым).
Родитель:
```html
<CascadingValue Value="@ThemeColor">
    <ChildComponent />
</CascadingValue>
```
Потомок:
```csharp
[CascadingParameter] public string ThemeColor { get; set; }
```

### 4. Как в родительском компоненте обработать событие дочернего компонента?
Родитель подписывается на `EventCallback` дочернего компонента.
```html
<Child OnClick="ParentMethod" />
```

### 5. Для чего используется метод StateHasChanged?
Он принудительно уведомляет Blazor о том, что состояние компонента изменилось и требуется повторный рендеринг. Это необходимо, когда изменения происходят асинхронно или вне стандартных обработчиков событий UI (например, при получении сообщения через SignalR или по таймеру).

### 6. Как работает компонент `<AuthorizeView/>`?
Он отображает свое содержимое в зависимости от того, авторизован пользователь или нет. Имеет секции `<Authorized>`, `<NotAuthorized>` и `<Authorizing>`. Он использует `AuthenticationStateProvider` для получения текущего состояния.

### 7. Каковы преимущества использования SignalR перед обычными HTTP запросами?
- **Двусторонняя связь**: Сервер может сам инициировать отправку данных клиенту (Push).
- **Низкая задержка**: Использует постоянное соединение (WebSockets), что исключает накладные расходы на установку соединения для каждого запроса.
- **Эффективность**: Меньше заголовков и трафика по сравнению с HTTP polling.

### 8. Как организовать групповое взаимодействие в SignalR?
Используя методы `Groups.AddToGroupAsync` и `Groups.RemoveFromGroupAsync` на сервере. Отправка сообщения группе: `Clients.Group("GroupName").SendAsync(...)`.

### 9. Как ограничить доступ неавторизованных пользователей к страницам Blazor?
Используя атрибут `[Authorize]` в компоненте страницы (`@attribute [Authorize]`).

### 10. Какие механизмы обеспечивают безопасность при работе с SignalR?
- **Аутентификация**: SignalR поддерживает стандартную аутентификацию ASP.NET Core (JWT, Cookies).
- **Авторизация**: Можно использовать атрибут `[Authorize]` на методах Хаба или на всем Хабе.
- **CORS**: Настройка Cross-Origin Resource Sharing для разрешения подключений только с доверенных доменов.
