# Лабораторная работа №3: Работа с данными

## Теория
В этой работе я научился передавать данные из контроллера в представление, используя модели и сервисы.

### Ключевые понятия
- **Entity (Сущность)**: Класс, представляющий объект реального мира (например, `Dish`, `Category`).
- **Service (Сервис)**: Класс, содержащий бизнес-логику и методы доступа к данным.
- **Dependency Injection (DI)**: Механизм внедрения зависимостей. Я регистрировал сервисы в контейнере DI и внедрял их в контроллер.
- **Model Binding**: Автоматическое сопоставление параметров HTTP-запроса с параметрами метода контроллера.

## Ход работы
1.  **Структура проекта**:
    - Создал отдельный проект `CPPR.Domain` для сущностей и общих моделей.
    - Добавил сущности `Dish` (Блюдо) и `Category` (Категория).
    - Создал модели `ResponseData<T>` (обертка для ответа) и `ListModel<T>` (для списков с пагинацией).
2.  **Сервисы**:
    - Реализовал `MemoryCategoryService` для хранения категорий в памяти.
    - Реализовал `MemoryProductService` для хранения списка блюд.
    - В `MemoryProductService` добавил логику фильтрации по категории и пагинации.
    - Зарегистрировал сервисы в `Program.cs` через метод расширения `RegisterCustomServices`.
3.  **Контроллер**:
    - Создал `ProductController`.
    - Внедрил в него `IProductService` и `ICategoryService`.
    - Реализовал метод `Index`, который принимает параметры `category` и `pageNo`.
4.  **Представление**:
    - Создал `Index.cshtml` для отображения каталога.
    - Использовал Bootstrap Cards для красивого вывода товаров.
    - Добавил выпадающий список для выбора категории.
    - Реализовал пагинацию (Bootstrap Pagination).
5.  **Конфигурация**:
    - Добавил настройку `ItemsPerPage` в `appsettings.json` для управления количеством элементов на странице.

## Использованные инструменты
